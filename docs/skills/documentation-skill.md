# Documentation Skill

Use this skill for README files, project docs, architecture notes, and other
GitHub-facing written material.

## Writing Rules

- Write GitHub-facing documentation in English.
- Keep documents concise, practical, and specific to their purpose.
- Avoid marketing claims, repeated background paragraphs, and speculative detail.
- Explain AoTune from the long-term multi-workspace product perspective.
- Do not make the naming explanation depend on the current Japanese Lyrics
  Learning MVP.
- Keep all five workspaces visible and conceptually equal while accurately
  identifying implementation status.
- Mention Yorushika only as personal mood and aesthetic inspiration. Never imply
  official affiliation, endorsement, or ownership.
- Clearly distinguish current behavior from planned architecture.
- Keep links and commands accurate relative to the repository.

## Maintenance

When behavior changes, update the smallest authoritative document instead of
copying the same explanation into multiple files. Check related documentation
for contradictions and stale phase labels.
